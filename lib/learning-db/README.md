# 学习型数据库 - 产品质量飞轮

## 核心理念
**这套数据库不是为了"存更多简历"，而是为了：**
- 形成标签库
- 形成改写库
- 形成反馈库
- 形成模式库
- 最终反哺诊断质量、改写质量和 deep 报告质量

## 三层架构

### 1. 原始生产数据层（已有）
- `UploadedFile` - 用户上传的文件
- `DiagnoseSession` - 诊断会话
- `DiagnoseReport` - 诊断报告
- `UsageRecord` - 使用记录

**作用**：支撑核心产品功能，记录基础交互。

### 2. 服务沉淀层（新增）
- `Lead` - 线索来源
- `ServiceCase` - 服务案例
- `CaseSnapshot` - 案例快照
- `DiagnosisLabel` - 诊断标签
- `RewritePair` - 改写对
- `FeedbackEvent` - 反馈事件

**作用**：记录完整的服务流程，从线索到交付到反馈。

### 3. 知识学习层（新增）
- `KnowledgePattern` - 知识模式
- `PatternEvidence` - 模式证据

**作用**：从服务案例中沉淀可复用的业务模式。

## 数据如何提升产品质量

### 1. UploadedFile / parse info → 提升 PDF 解析策略
- 积累不同格式、不同来源的简历文件
- 分析解析成功/失败案例，优化文本提取算法
- 形成文件处理最佳实践

### 2. DiagnosisLabel → 提升 main_judgment 与 issue ranking 准确度
- 人工专家标注 vs AI 诊断结果对比
- 积累高质量风险评级（ATS风险、HR风险、方向匹配度）
- 训练 AI 识别更细微的问题维度
- 减少误判，提高诊断置信度

### 3. RewritePair → 提升 rewrite pattern 质量
- 积累"原文-改写后"对照对
- 分析不同问题类型（技术栈老旧、缺乏量化、关键词缺失）的最佳改写策略
- 形成不同岗位（前端、后端、产品、运营）的改写模式
- 为 AI 改写提供高质量训练数据

### 4. FeedbackEvent → 提升建议采纳率与有效性判断
- 跟踪用户对改写建议的采纳情况
- 收集交付后反馈（满意度、面试数、Offer数）
- 验证改写建议的实际效果
- 识别高价值建议模式，优化推荐策略

### 5. KnowledgePattern → 反哺 corpus / rules / prompts
- 从多个成功案例中提取共性模式
- 形成可复用的诊断规则库
- 优化现有语料库（Corpus）质量
- 生成更精准的提示词（prompts）
- 为新产品功能（如面试风险预测、职业路径规划）提供数据支持

## 产品质量飞轮闭环

```
    线索进入 → 服务开始 → 诊断完成 → 人工标注
        ↓          ↓          ↓          ↓
    渠道优化   流程优化   准确度提升   标签积累
        ↓          ↓          ↓          ↓
    改写完成 → 反馈收集 → 模式沉淀 → 产品增强
        ↓          ↓          ↓          ↓
    改写库     反馈库     模式库     智能提升
```

### 具体提升路径

1. **短期（1-2周）**：通过 `ServiceCase` + `attachSessionToCase` 关联现有诊断数据，开始积累案例基础。

2. **中期（1-2月）**：通过人工标注 (`DiagnosisLabel`) 和改写 (`RewritePair`) 积累核心知识资产，提升 AI 诊断与改写的准确性。

3. **长期（3-6月）**：通过反馈跟踪 (`FeedbackEvent`) 和模式沉淀 (`KnowledgePattern`)，形成产品的专有知识库，建立竞争壁垒。

## 专有知识资产形成

### 标签库
- 高频简历问题标签体系
- 风险评级标准（ATS/HR/方向匹配）
- 岗位家族特征库

### 改写库
- 不同问题类型的改写策略
- 不同岗位的表述风格
- 量化成果的表达模式

### 反馈库
- 用户采纳行为模式
- 服务效果验证数据
- 满意度驱动因素

### 模式库
- 诊断模式：如何识别特定问题
- 改写模式：如何解决特定问题
- 风险模式：如何预测投递结果
- 路径模式：如何规划职业发展

## 技术实施价值

### 对现有系统
- **诊断模块**：获得更准确的判断依据
- **改写模块**：获得更优质的训练数据
- **报告模块**：获得更丰富的案例支撑

### 对新功能开发
- **智能推荐**：基于历史案例推荐最优改写方案
- **风险预测**：基于相似案例预测简历投递成功率
- **职业规划**：基于成功路径提供发展建议

## 数据安全与合规

- 尊重用户授权：`consentSaveRaw` 和 `consentUseAnonymized` 确保透明控制
- 数据最小化：默认不存储原始简历，需要时存储匿名化版本
- 定期清理：遵循数据保留策略，平衡价值与隐私

## 后续开发建议

1. **管理界面**：简单的 admin 后台用于人工标注和案例查看
2. **自动化分析**：周度/月度自动生成模式发现报告
3. **集成优化**：将沉淀的模式自动应用到诊断和改写流程
4. **API 开放**：未来可开放部分匿名数据供研究使用

---

**核心价值**：每一次服务不仅解决用户问题，同时提升产品智能，形成越用越强的正向循环。
