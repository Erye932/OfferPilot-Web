# 我的工作手册（非工程师版）

欢迎使用 OfferPilot 服务运营系统！这份手册告诉你如何使用这个系统，**不需要懂任何技术**。

## 🎯 系统是做什么的？

你每做一单服务，系统帮你：
1. **自动记录**案例到数据库（不用手写）
2. **自动分析**数据，发现规律
3. **自动生成**周报，发给 AI 复盘
4. **自动沉淀**有效模式，持续提升诊断质量

你只需要：**填模板 + 跑命令**。

---

## 📁 文件结构（你只需要关心这些）

```
offerpilot-web/
├── ops/                          # 你的工作目录
│   ├── templates/                # 模板目录
│   │   └── case-template.md      # 案例模板（复制这个）
│   ├── cases/                    # 案例目录（放填好的案例）
│   │   └── demo-case.md          # 示例案例（参考）
│   ├── reports/                  # 报告目录（自动生成）
│   └── patterns/                 # 模式目录（自动生成）
└── （其他技术文件不用管）
```

## 📝 每天怎么做（新用户来了）

### 第1步：创建案例文件
1. 打开 `ops/templates/case-template.md`
2. 复制整个文件内容
3. 在 `ops/cases/` 目录新建文件，例如：`2026-04-03-小红书-小明.md`
4. 粘贴模板内容

### 第2步：填写内容
按模板逐项填写，**注意**：
- 带反引号 `` ` ` `` 的选项必须用里面的值，例如：`` `xiaohongshu` ``
- 不知道的留空
- 重点填写：
  - **基本信息**（来源、服务类型、岗位）
  - **我的最终判断**（最致命问题、风险等级）
  - **改写记录**（原文、改后、是否采纳）

### 第3步：运行入库命令
打开终端（命令提示符），进入项目目录，运行：
```bash
npm run sync:cases
```

### 第4步：检查结果
命令运行后会显示：
- ✅ 成功：案例已保存
- ⚠️ 警告：缺少某些字段（可忽略）
- ❌ 失败：格式错误（按提示修复）

**这就完成了！** 案例已自动存入数据库。

## 🔄 每单交付后怎么做

### 交付时补全信息
1. 打开对应的案例文件
2. 填写 **"5. 交付后反馈"** 部分
3. 运行 `npm run sync:cases` 更新数据库

### 7天后补全信息
1. 打开案例文件
2. 填写 **"6. 回访记录 - 7 天"** 部分
3. 运行 `npm run sync:cases`

### 30天后补全信息
1. 打开案例文件
2. 填写 **"6. 回访记录 - 30 天"** 部分
3. 运行 `npm run sync:cases`

**注意**：同一个案例文件可以多次运行 `npm run sync:cases`，系统会自动更新。

## 📊 每周怎么做（生成周报）

### 第1步：运行周报命令
每周五（或任何时间）运行：
```bash
npm run report:weekly
```

### 第2步：查看报告
命令会自动生成报告文件：
```
ops/reports/weekly-2026-04-03.md
```

用记事本或VSCode打开这个文件查看。

### 第3步：发给 AI 分析
复制整个报告内容，发给 Claude/ChatGPT 等 AI，提问：
> "这是 OfferPilot 本周服务数据报告，请分析：1. 主要问题是什么？2. 如何改进服务？3. 哪些模式值得固化？"

根据 AI 建议调整下周工作重点。

## 🧠 知识沉淀（可选）

## ❓ 常见问题

### Q1：数据库是什么？需要我维护吗？
**不需要**。数据库是自动管理的，你只需要运行命令，系统会自动读写。

### Q2：案例文件命名有要求吗？
没有严格限制，建议：`日期-渠道-昵称.md`，方便查找。

### Q3：填错了怎么办？
直接修改案例文件，重新运行 `npm run sync:cases`。

### Q4：数据安全吗？
案例文件在本地，数据库也在本地。**不要**上传包含个人信息的案例文件到网上。

### Q5：需要安装什么软件？
已经安装好了：Node.js、数据库。你只需要运行命令。

### Q6：命令运行出错怎么办？
检查：
1. 是否在项目目录下运行
2. 案例文件格式是否正确（特别是反引号）
3. 终端错误信息会提示具体问题

6. **满意度提高** → 更多案例
7. **更多案例** → 更多数据...

形成**数据驱动改进**的正循环。
