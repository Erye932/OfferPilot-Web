# OfferPilot Agents

## 当前裁决

这个目录只保留有真实外部输入、可验证数据来源、或后续能服务真实反馈闭环的脚本。

已删除的脚本不是“能力资产”，而是会误导判断的模拟自动化：

- `orchestrator.ts`：调度的是一组模拟脚本，会制造“系统在自动赚钱”的错觉。
- `pain-radar.ts`：使用预设痛点库和随机热度，不是真实平台扫描。
- `content-factory.ts`：生成多平台营销文案，存在虚假背书、平台混发和 AI 味风险。
- `content-strategist.ts`：生成强营销投放策略，包含不适合当前账号的互动套路和虚假案例表达。
- `monetization.ts`：生成模拟交易、模拟转化率和模拟收入日报，不能用于商业决策。
- `data-engine.ts`：使用 mock 简历解析，不是真实 OCR/PDF/文本处理链路。
- `evolution.ts`：使用 mock 用户反馈，还可能改写知识库规则，风险高于收益。

## 当前保留

- `pain-radar-tavily.ts`

保留原因：它至少接入真实搜索 API，并带缓存；虽然仍需人工判断，但不是纯模拟数据。

使用前提：

1. `.env.local` 中存在 `TAVILY_API_KEY`。
2. 搜索结果只作为选题线索，不作为“真实需求已经验证”的证据。
3. 输出文案不能直接发布，必须人工按当前平台定位重写。

运行方式：

```bash
npx tsx agents/pain-radar-tavily.ts --query="简历 求职 2026"
```

## 不再做的事

- 不再用模拟收入日报判断今天该做什么。
- 不再用随机痛点热度决定发布方向。
- 不再用自动生成文案直接发小红书、闲鱼、知乎、即刻。
- 不再开启 `orchestrator` 挂机自动调度。

## 当前真实闭环

真实有效的数据只来自：

1. 闲鱼：曝光、浏览、想要、私信、成交。
2. 小红书：浏览、点赞、收藏、评论、私信。
3. OfferPilot：真实用户提交、真实诊断结果、真实反馈。
4. 已付费案例：回收类小工具 350 元成交记录与交付过程。

后续如果要恢复 Agent，必须满足一个标准：

> 输入是真实数据，输出能指导一个具体动作，并且结果能被复盘验证。
