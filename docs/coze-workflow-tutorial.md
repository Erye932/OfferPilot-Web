# Coze 工作流配置教程 - OfferPilot 简历诊断

> 这是一份手把手教程，你只需要按顺序操作，不需要理解原理。

---

## 第一步：创建新 Bot

1. 打开 [coze.cn](https://www.coze.cn)
2. 点击左侧「工作空间」→「创建 Bot」
3. 填写：
   - **Bot 名称**：`简历诊断助手`
   - **Bot 简介**：`上传简历，AI帮你诊断问题并给出改写建议`
4. 点击「确认」

---

## 第二步：创建工作流

1. 在 Bot 编辑页面，点击左侧「工作流」标签
2. 点击「创建工作流」
3. 填写：
   - **工作流名称**：`resume_diagnose`
   - **描述**：`简历诊断主流程`
4. 点击「确认」

---

## 第三步：配置输入变量

在工作流编辑器左侧「输入」区域，添加以下变量：

| 变量名 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `resume_text` | String | 是 | - | 用户粘贴的简历文本 |
| `target_role` | String | 是 | - | 目标岗位名称 |
| `jd_text` | String | 否 | "" | 职位描述（可选） |

---

## 第四步：添加节点 - 创建诊断任务

### 4.1 添加 HTTP 请求节点

1. 从左侧节点库拖入「HTTP 请求」节点
2. 命名为：`创建任务`

### 4.2 配置节点参数

| 参数 | 值 |
|------|-----|
| **请求方式** | POST |
| **URL** | `https://offerpilot-web.vercel.app/api/diagnose/tasks` |
| **Headers** | `Content-Type: application/json` |
| **Body 类型** | JSON |
| **Body 内容** | 见下方 |

**Body 内容**（直接复制）：
```json
{
  "resume_text": "{{input.resume_text}}",
  "target_role": "{{input.target_role}}",
  "jd_text": "{{input.jd_text}}",
  "tier": "free"
}
```

---

## 第五步：添加节点 - 提取 task_id

### 5.1 添加变量赋值节点

1. 拖入「变量赋值」节点
2. 命名为：`提取task_id`

### 5.2 配置变量

| 变量名 | 值表达式 |
|--------|----------|
| `task_id` | `{{创建任务.result.task_id}}` |

---

## 第六步：添加节点 - 轮询任务状态

### 6.1 添加循环节点

1. 拖入「循环」节点
2. 命名为：`轮询任务状态`

### 6.2 配置循环参数

| 参数 | 值 |
|------|-----|
| **循环类型** | 条件循环 |
| **最大次数** | 20 |
| **循环间隔** | 15 秒 |

### 6.3 在循环内添加 HTTP 请求节点

在循环内部添加一个 HTTP 请求节点：

| 参数 | 值 |
|------|-----|
| **节点名称** | `查询状态` |
| **请求方式** | GET |
| **URL** | `https://offerpilot-web.vercel.app/api/diagnose/tasks/{{task_id}}` |

### 6.4 配置循环退出条件

**继续循环条件**：
```
{{查询状态.result.status}} != "done" AND {{查询状态.result.status}} != "failed"
```

---

## 第七步：添加节点 - 条件分支

### 7.1 添加条件分支节点

1. 拖入「条件分支」节点
2. 命名为：`判断结果`

### 7.2 配置分支条件

**分支 1 - 成功**：
- 条件：`{{查询状态.result.status}} == "done"`
- 动作：进入「获取报告」节点

**分支 2 - 失败**：
- 条件：`{{查询状态.result.status}} == "failed"`
- 动作：进入「输出错误」节点

**分支 3 - 超时**：
- 条件：`{{轮询任务状态.loop_count}} >= 20`
- 动作：进入「输出超时」节点

---

## 第八步：添加节点 - 获取报告

### 8.1 添加 HTTP 请求节点（在「成功」分支下）

| 参数 | 值 |
|------|-----|
| **节点名称** | `获取报告` |
| **请求方式** | GET |
| **URL** | `https://offerpilot-web.vercel.app/api/diagnose/report/{{查询状态.result.report_id}}` |

---

## 第九步：添加节点 - 输出结果

### 9.1 添加输出节点（在「获取报告」后）

1. 拖入「输出」节点
2. 命名为：`输出诊断结果`

### 9.2 配置输出变量

| 变量名 | 值表达式 |
|--------|----------|
| `main_judgment` | `{{获取报告.result.main_judgment}}` |
| `core_issues` | `{{获取报告.result.core_issues}}` |
| `minor_suggestions` | `{{获取报告.result.minor_suggestions}}` |
| `rewrite_examples` | `{{获取报告.result.rewrite_examples}}` |
| `excellent_score` | `{{获取报告.result.excellent_score}}` |

---

## 第十步：添加节点 - 错误处理

### 10.1 添加输出节点（在「失败」分支下）

| 变量名 | 值表达式 |
|--------|----------|
| `error` | `{{查询状态.result.error_message}}` |

### 10.2 添加输出节点（在「超时」分支下）

| 变量名 | 值 |
|--------|-----|
| `error` | `"任务处理超时，请稍后再试"` |
| `task_id` | `{{task_id}}` |

## 常见问题

### Q1: 循环节点找不到？
Coze 的循环节点可能在「高级节点」里，或者叫「迭代器」。

### Q2: 变量表达式怎么写？
Coze 用 `{{节点名.result.字段名}}` 格式引用上游节点的输出。

### Q3: 测试时报 404？
检查 URL 是否正确，确保没有多余的空格或换行。

### Q4: 任务一直 queued？
诊断任务需要 worker 触发。如果队列里的任务没有被处理，可以手动触发：
```
POST https://offerpilot-web.vercel.app/api/internal/diagnose-worker
Body: {"task_id": "你的task_id"}
```
